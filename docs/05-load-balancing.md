# 05 – Load Balancing with nginx

## Overview

nginx acts as both the **web server** (on `webservers` hosts) and a **reverse proxy / load balancer** (on the `loadbalancer` host).

```
Client → :80 (LB) → upstream { web1:80, web2:80, ... }
```

## Algorithm

`least_conn` — each new request is forwarded to the backend with the fewest active connections. This is more adaptive than round-robin when requests vary in duration.

## Generated upstream config (`lb.conf.j2`)

```nginx
upstream backend {
    least_conn;
    server 192.168.1.10:80;   # web server 1
    server 192.168.1.11:80;   # web server 2
    # ... auto-generated by Ansible from groups['webservers']
}

server {
    listen 80;

    location / {
        proxy_pass         http://backend;
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_connect_timeout 5s;
        proxy_read_timeout    30s;
    }

    location /health {
        return 200 "OK\n";   # health check endpoint
    }
}
```

The upstream server list is **dynamically rendered** by Ansible — no manual editing needed when adding or removing nodes.

## Adding a new web server

1. Add its IP to the `WEB_HOSTS` GitHub Actions variable (JSON array).
2. Push any commit to `main`.
3. Terraform regenerates the inventory → Ansible installs nginx on the new node **and** updates the LB upstream block automatically.

## Health check

```bash
curl http://<lb-ip>/health
# → OK
```

## Testing load distribution

```bash
for i in $(seq 1 10); do curl -s http://<lb-ip>/ | grep -o 'Server [0-9]*'; done
```

You should see requests distributed across the backend nodes.
